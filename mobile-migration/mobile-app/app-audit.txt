================================================================================
  1. PROJECT OVERVIEW & TECH STACK
================================================================================

Application: Cross-platform portfolio management & financial analysis app.
Platforms:   Web (Metro bundler), iOS (Expo EAS), Android (Expo EAS).
Framework:   Expo SDK 54 + React Native 0.81.5 + Expo Router v6
Backend:     FastAPI on port 8004 (separate repo)
Version:     1.0.0
Language:    TypeScript 5.9, strict mode

Core features:
  • Portfolio overview with hero KPIs, charts, per-portfolio cards
  • Holdings management with 18-column sortable tables
  • Portfolio analysis with performance metrics (TWR, MWRR, Sharpe, Sortino)
  • Transaction CRUD with bulk import (KFH trade statements)
  • Deposit tracking with portfolio filters
  • Dividend and bonus share tracking with yearly charts
  • Fundamental analysis (7 sub-tabs: stocks, statements, comparison,
    metrics, growth, scoring, valuations) + decision panel
  • AI-powered financial extraction (Gemini 2.5 Flash / Pro)
  • AI Financial Intelligence panel with prompt library
  • Time Value of Money planner (CFA-level: FV/PV/Savings/Loan/Goal)
  • Personal Finance Manager (balance sheet, income/expense, ratios)
  • Portfolio tracker (time-series snapshots with daily movement)
  • Trading overview (12 summary cards, 19-column sortable table)
  • Trade simulator ("What If" scenario analysis)
  • Securities master list management
  • Boursa Kuwait live market data (indices, mood bar, risers/drops, sectors)
  • News feed (cursor-based infinite scroll, categories, sentiment)
  • Alert rules (price/portfolio alerts with push notifications)
  • Kuwait Market Pulse (local insights panel)
  • Trending stocks (anonymized social signals)
  • Strategy selector (Dividend / Growth / Islamic)
  • Portfolio health card (Healthy / Attention / At Risk)
  • Sharia compliance screening (Musaffa.com-based)
  • Dividend income projector (GCC schedules, bonus shares)
  • Decision engine (Buy / Hold / Sell insights from scores)
  • Daily WhatsApp/email briefing (opt-in)
  • Data integrity checks (cash/position/snapshot/anomaly)
  • Backup & restore
  • Admin dashboard (users, activities, registration trends)
  • PDF export (Growth, Metrics, Valuation, Yield reports)
  • CSV/Excel export for fundamental analysis
  • Google OAuth + email/password authentication
  • Progressive disclosure (new → intermediate → power user)
  • Expertise level selection (normal / intermediate / advanced)
  • i18n support (English + Arabic, RTL-ready)
  • Dark & light theme (Streamlit-matching palettes)
  • Responsive design (phone < 768px, tablet 768–1024px, desktop > 1024px)
  • Onboarding flow for first-time users
  • Push notifications (price updates, news alerts)

================================================================================
  6. AUTH SYSTEM
================================================================================

AUTH FLOW (3-layer guard):
──────────────────────────

  Layer 1: app/index.tsx
    • Checks Zustand auth token + onboarding flag
    • Redirects unauthenticated → /(auth)/login
    • Redirects first-time → /(onboarding)/welcome
    • Redirects authenticated → /(tabs)/

  Layer 2: app/(tabs)/_layout.tsx
    • Tabs layout guard (declarative <Redirect> + useEffect fallback)

  Layer 3: app/(tabs)/index.tsx
    • Overview screen guard (declarative <Redirect>)

AUTHENTICATION METHODS:
───────────────────────

  1. Email/Password (login + register)
     • Zod-validated forms (react-hook-form + @hookform/resolvers)
     • Password strength meter on register
     • Password requirements enforced in validationSchemas.ts
     • POST /api/v1/auth/login → JWT access + refresh tokens

  2. Google OAuth
     • Web: implicit flow via expo-auth-session + expo-web-browser
     • Native: OS-native via @react-native-google-signin
     • Backend validates Google ID token, creates/finds user
     • POST /api/v1/auth/google → JWT tokens

  3. Forgot Password (676 lines)
     • 3-step OTP flow: email → OTP verification → new password

TOKEN MANAGEMENT (tokenStorage.ts — 140 lines):
────────────────────────────────────────────────

  Platform-aware:
    Web    → sessionStorage (access), localStorage (refresh)
    Native → expo-secure-store (encrypted)

  JWT expiry parsing (isTokenExpired):
    • Decodes JWT payload (base64 → JSON)
    • Checks exp claim with 60s buffer

  Proactive refresh (client.ts):
    • Before each request, checks if token is expired
    • If expired, silently calls /api/v1/auth/refresh
    • De-duplicates concurrent refresh calls

  401 interceptor (client.ts):
    • On 401 response, attempts silent refresh
    • Queues pending requests until refresh completes
    • On refresh failure → full logout

ERROR HANDLING (authErrors.ts — 237 lines):
────────────────────────────────────────────

  • Maps raw API/network errors → AuthError objects
  • Fields: code, message, severity (info/warning/error/critical)
  • User-friendly messages for common scenarios
  • Sentry-compatible error codes for analytics

================================================================================
  21. DESIGN PATTERNS & KEY DECISIONS
================================================================================

  1. FILE-BASED ROUTING
     Expo Router v6 maps filesystem → routes. Groups (auth), (onboarding),
     (tabs) organize without affecting URLs.

  2. FEATURE-SLICED DESIGN
     Fundamental Analysis uses a self-contained feature module under
     src/features/. Types, styles, hooks, and components co-located.

  3. DOMAIN-ORGANIZED API
     services/api/ splits endpoints by domain (auth, portfolio, transactions,
     analytics, extraction). Shared types in types.ts, shared client in client.ts.
     Additional domain services (market, news, alerts, briefings, insights,
     social signals) in dedicated subdirectories.

  4. QUERY KEY FACTORIES
     Each query hook file defines its own key factory for consistent
     cache invalidation. Pattern: [domain, action, params].
     Cursor-based infinite queries for news feed.

  5. RESPONSIVE-FIRST
     useResponsive() hook provides breakpoints, spacing, fonts, and
     layout helpers. All screens adapt to phone/tablet/desktop.

  6. PROGRESSIVE DISCLOSURE
     Tab visibility scales with user experience (new → normal → power).
     Expertise level (normal/intermediate/advanced) gates feature access.
     Reduces complexity for new users.

  7. PLATFORM-AWARE PERSISTENCE
     tokenStorage.ts and auth store abstract web (localStorage/sessionStorage)
     vs native (expo-secure-store) behind a common interface.
     MMKV for high-performance key-value storage (native).

  8. SKELETON-FIRST LOADING
     PageSkeletons.tsx provides loading states for ALL screens.
     No "flash of empty" — skeleton content matches real layout.

  9. TOAST FEEDBACK
     ToastProvider provides non-blocking notifications for mutations.
     useTransactionMutations adds haptic feedback on native.

  10. THEME SYSTEM
      ThemePalette interface enforced across all components.
      Two palettes (light/dark) matching legacy Streamlit CSS.
      Zustand store persists user preference.

  11. ERROR BOUNDARIES
      AppErrorBoundary at root catches unhandled errors.
      Feature-level ErrorBoundary in fundamental analysis shared.tsx.
      ErrorScreen provides recovery UI.

  12. FORM ARCHITECTURE
      react-hook-form + Zod validation + FormField components.
      FormScreen template for consistent form layout.

  13. CHART SYSTEM
      SVG-based charts via react-native-svg.
      Chart animation via useChartAnimation (react-native-reanimated).
      5 chart types: area, dual-area, donut, bar, waterfall.
      Interactive tooltips with crosshair.

  14. DRAG & DROP
      @dnd-kit (web only) for statement line item reordering.
      Graceful fallback on native (manual reorder).

  15. INTERNATIONALIZATION                                        [NEW]
      i18next + react-i18next with expo-localization.
      English + Arabic translations. RTL-ready layout.

  16. NOTIFICATION SYSTEM                                         [NEW]
      expo-notifications for local/push alerts.
      Price updates, news alerts, alert rule triggers.
      Dedup logic to prevent notification spam.

  17. STRATEGY-BASED INVESTMENT                                   [NEW]
      3 investment strategy templates (Dividend/Growth/Islamic).
      Scoring weights, sector preferences, risk tolerance.
      Strategy alignment shown on dashboard.

  18. SHARIA COMPLIANCE                                          [NEW]
      Musaffa.com-based screening criteria.
      ShariaBadge component with expandable screening details.
      Integrated into holdings view.

  19. ANIMATED TAB BAR                                           [NEW]
      Custom AnimatedTabBar with spring physics, scale bounce,
      and sliding indicator. Replaces default bottom tabs on mobile.
