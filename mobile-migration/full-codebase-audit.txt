  5. APP ENTRY POINT & MIDDLEWARE
================================================================================

app/main.py (196 lines)

  Startup Configuration:
    • Database existence check (requires dev_portfolio.db)
    • Schema initialization via ensure_all_tables()
    • Extraction job recovery (fundamental analysis)
    • yf_ticker backfill for existing stocks
    • Production security audit (SECRET_KEY, CRON_SECRET_KEY, CORS)

  Middleware Stack (in order):
    1. SecurityHeadersMiddleware — X-Frame-Options, X-Content-Type-Options, etc.
    2. RequestSizeLimitMiddleware — Limits payload to 50 MB
    3. CORSMiddleware — Dev origins vs prod origin regex matching
    4. PrivateNetworkAccessMiddleware — RFC1918 preflight support

  Exception Handlers:
    • RateLimitExceeded → via slowapi
    • APIError → custom structured responses
    • Exception → unhandled error logging

  Routers Included:
    • v1 (primary): /api/v1/* namespace with 20 sub-routers
    • Legacy: /auth, /portfolio, /cron (backward-compat)

  Health Endpoints:
    • GET /health — Status + db_connected, db_mode, environment
    • GET /api/health — Same (for Render routing)
    • GET /health/tables — Diagnostic: lists all database tables

================================================================================
  31. ARCHITECTURE PATTERNS & DESIGN DECISIONS
================================================================================

  Backend:
    • Layered Architecture: Routes → Services → ORM/Raw SQL → Database
    • Dual DB Support: SQLite (dev WAL mode) / PostgreSQL (prod)
    • Schema-First: ensure_all_tables() creates schema idempotently
    • Soft-Delete Pattern: is_deleted flag on transactions & deposits
    • Audit Trail: All security-relevant actions logged to audit_log
    • Rate Limiting: slowapi per-IP, per-endpoint limits
    • Worker Safety: File-based lock for single scheduler instance
    • CFA/IFRS Compliance: WAC engine, TWR, MWRR, Modified Dietz
    • Dual API Versions: Legacy (/auth, /portfolio) + v1 (/api/v1/*)

  Frontend:
    • State Management: Zustand (auth, theme, prefs) + React Query (server)
    • Code Organization: Expo Router file-based + domain features (src/)
    • Form Pattern: react-hook-form + Zod schemas + step wizards
    • Responsive: 3-tier breakpoint system (phone/tablet/desktop)
    • Offline First: react-native-mmkv + offline queue in API client
    • Error Handling: ErrorBoundary + Sentry + toast notifications
    • Lazy Loading: Heavy screens lazy-loaded via src/screens/
    • New Architecture: React Native new arch enabled (Fabric + TurboModules)
    • Typed Routes: Expo Router typed routes for compile-time safety
